**日期**: 2026-06-04
**任务目标**: FAR < 0.002% (1/50000) 时 FFR < 3%

## 3. 各版本详细分析

### 3.1 V14 — 基线 (EER = 2.6776%)

**架构**: ResNet-18 (ImageNet预训练) + GAP + Linear → 512维嵌入
**训练**: SimCLR 80ep（冻结backbone）+ ArcFace 120ep
**推理**: 双极性TTA，余弦相似度

**问题**: 无SWA导致过拟合；无分数归一化；简单模板选择

---

### 3.2 V15 — SWA引入 (EER = 2.40%)

**改进**: 在V14基础上加入SWA（最后20个epoch权重平均）
**结论**: SWA对小数据集有效，零额外推理开销

---

### 3.3 V16 — 复杂架构尝试 (EER = 2.5144%)

**架构**: ConvNeXt-Tiny (28.6M参数) + 17通道Gabor输入 + GeM池化 + 多尺度特征聚合

| 配置 | Backbone | 参数量 | EER |
|------|----------|--------|-----|
| Config A | ConvNeXt-Tiny | 28.6M | 2.5144% |
| Config C | ResNet-18 | 11.4M | 3.7870% |

**结论**: 28.6M参数在27类小数据集上过拟合，简单ResNet-18更优。

---

### 3.4 V17 — 分数归一化突破 (EER = 1.6416%)

**核心创新**: T-norm分数归一化 — 零训练开销的后处理方法

**架构**: ResNet-18 + GAP + Linear → 512维，SimCLR 80ep + ArcFace 140ep（SWA@110, label_smoothing=0.1）
**推理**: 4-view TTA + 3模型集成 + T-norm

**T-norm**: `t_score = (raw - mean(impostor_scores)) / std(impostor_scores)`

| 模板策略 | raw | Z-norm | T-norm | S-norm |
|----------|-----|--------|--------|--------|
| quality | 2.19% | 2.38% | 2.48% | 2.32% |
| random | 2.22% | 1.76% | **1.64%** | 1.96% |

**最佳(random_tnorm)工作点**: FFR=3%→FAR=0.37%, d-prime=5.62

**关键发现**: Random模板 + T-norm效果最好，Quality模板多样性不足反损害T-norm。

---

### 3.5 V18 — 未训练密集局部匹配 (EER = 1.6865%)

**思路**: 借鉴FLARE/DMD空间描述符，复用V17 layer3特征图(256, 14, 13)做dense匹配，无需重训。

| 变体 | EER |
|------|-----|
| dense_raw | 8.89% |
| global_tnorm | 2.07% |
| fused_a0.3_tnorm | **1.69%** |

**结论**: 全局训练的layer3特征不适合局部匹配(EER=8.89%)，**需要专门训练局部描述符**。

---

### 3.6 V18b — 训练局部描述符 (EER = 1.3341%)

**核心创新**: Layer3后加局部描述符头 Conv1x1(256→64)+BN+L2-norm，通过PositionContrastiveLoss联合训练

**模型架构**:
```
ResNet-18 backbone
├─ Layer3 (256, 14, 13) → local_head → 64维局部描述符
└─ Layer4 → GAP → 512维全局嵌入
```

**训练**: Phase 1 SimCLR+LocalCL (60ep) | Phase 2 ArcFace+LocalCL (120ep, SWA@100)
**联合损失**: `loss = ArcFace + 0.3 × PositionContrastiveLoss`

| 变体 | EER |
|------|-----|
| global_tnorm | 1.41% |
| fused_a0.3_tnorm | 1.34% |
| **fused_a0.5_tnorm** | **1.33%** |
| fused_a0.7_tnorm | 1.46% |

**最佳工作点**: FFR=3%→FAR=0.32%, d-prime=5.20

**关键发现**: 训练后局部描述符手指间极差从0.102缩小到0.036；联合训练也改善了全局嵌入(1.74→1.41%)。

---

### 3.7 V19 — RANSAC几何验证 (EER = 1.3294%)

**思路**: 参考DLG光学指纹代码，在V18b的14×13网格局部描述符上加RANSAC仿射验证+LPM。

| 变体 | EER |
|------|-----|
| ransac_inlier_tnorm | 19.56% |
| ransac_lpm_tnorm | 52.77% |
| v18b_fused_a0.5_tnorm | **1.33%** |

**RANSAC内点**: Genuine均值94.2, Impostor均值**67.2**, d-prime仅1.37

**失败原因**:
1. 14×13=182点网格太粗，RANSAC仿射约束太弱
2. 220×200图像太小（vs DLG的224×224光学指纹）
3. 固定均匀网格 vs DLG的注意力选点+多尺度NMS
4. **Impostor也能找到67个"几何一致"内点，几乎无区分度**

---

### 3.8 V20 — DELF注意力池化 (EER = 1.5967%) ⚠️ 退步

**思路**: 照搬DELF架构，去掉layer4，用注意力加权池化在layer3做全局嵌入。

| 变体 | V20 | V18b |
|------|-----|------|
| global_tnorm | **1.82%** | 1.41% |
| dense_topk_tnorm | **5.78%** | — |
| 最佳融合 | **1.60%** | **1.33%** |

**失败原因**: 去掉layer4导致参数量从11.5M降到3M，全局嵌入退步0.41%。
**但**: 注意力选点有效 — dense从8.05%降到5.78%，证明注意力机制本身有价值。

### 3.9 V21 — 混合架构 (EER = 1.1898%) ⬅ 当前最优

**核心思路**: 吸取V18b/V20教训，保留layer4全局嵌入 + 在layer3加DELF注意力做辅助训练

**模型架构**:
```
ResNet-18 backbone
├─ Layer3 (256, 14, 13)
│    ├─ attention: Conv1x1(256→128)→ReLU→Conv1x1(128→1)→Softplus
│    │    └─ 辅助路径: L2-norm(feat)×attention→sum→Linear(256→512)→辅助ArcFace
│    └─ local_head: Conv1x1(256→64)+BN+L2-norm → 局部描述符
└─ Layer4 → GAP → Linear(512→512)+BN → 主全局嵌入 → 主ArcFace
```

**参数量**: 11,616,065（V18b + 166K注意力模块）

**训练**:
- Phase 1 (60ep): SimCLR(主全局) + 0.3×SimCLR(注意力) + 0.5×LocalCL
- Phase 2 (120ep): ArcFace(主全局) + 0.3×ArcFace(注意力) + 0.3×LocalCL, SWA@100

**关键设计**: 注意力辅助ArcFace作为**多任务正则化**，迫使layer3学到更有空间判别性的特征，间接提升了layer4+GAP的全局嵌入。

| **全局** | **global_raw** | **1.1898%** ★ |
| 全局 | global_tnorm | 1.6605% |
| Dense(全部) | dense_all_tnorm | 8.01% |
| Dense(注意力选点) | dense_topk_tnorm | 6.79% |

### 3.10 V24 — 客户新数据(ysjz)加入训练 (2026-06-10/11) ⚠️ 无提升

**背景**: 客户上传两批新数据 `ysjz.zip`(原图) 与 `ysjz_denoised.zip`(降噪后)，5人×22个手指文件夹（含左手指、新人yjx），每folder约100帧，110×100 BMP，格式与老数据一致。

**身份检验（训练前用V21模型验证）**:
- 新数据同名手指（dy_R0/lwh_R0/zyh_R0等）与老数据同名类的嵌入相似度仅 **0.2~0.4（impostor水平）**，远低于genuine水平(0.6~0.99)
- 多个不同新文件夹"最像"同几个老类（wtp_dhy_R0/wtp_dy_R1/btp_ssh_R0），是典型域偏移特征
- 视觉对比：新数据噪声显著更大、有横条纹伪影；降噪版脊线更清晰
- **结论：新数据作为独立新类加入，不与老类合并**
- 异常: `ssh_R0_failure`(19帧)排除；`SSH_L2-half`(63帧)保留；SSH_L0降噪版与btp_ssh_R0相似度0.81(可疑，建议向客户确认是否同一手指)

**实验设计**: V21架构/超参/评估协议完全不变（老27类，70/30分割，每类20模板，3 seeds），新数据21类仅加入训练（27→48类），EER与V21直接可比。两个对照组：
- **V24a** = V21 + ysjz原图 (`models_v24a/`)
- **V24b** = V21 + ysjz降噪 (`models_v24b/`)

**结果**:

| 指标 | V21基线 | V24a (+原图) | V24b (+降噪) |
|------|---------|-------------|-------------|
| 最佳EER | **1.1898%** (global_raw) | 1.2608% (fused_w_a0.7_tnorm) | 1.8829% (fused_w_a0.7_tnorm) |
| global_raw | **1.1898%** | 1.7102% | 3.1673% |
| fused_weighted_a0.7_tnorm | 1.28% | **1.2608%** | 1.8829% |
| d-prime (最佳变体) | **6.33** | 4.83 | 4.71 |
| FFR=3% → FAR | **0.26%** | 0.2744% | 0.7711% |
| FAR=0.01% → FFR | 15.13% | **11.44%** | 17.59% |

**发现**:
1. **两个arm都未超过V21基线** — 新数据域偏移太大，混入训练拉低了老协议上的全局嵌入质量（global_raw: 1.19→1.71/3.17%）
2. **降噪版伤害远大于原图版**(1.88% vs 1.26%) — 推测：降噪把新数据拉近老域分布（如SSH_L0_denoised↔btp_ssh_R0相似度0.81），迫使ArcFace在老域附近强行分开"长得像老类"的新类，扭曲了老类嵌入空间；而原图域远，干扰反而小
3. **T-norm重新变得有效**（V24a global: 1.71→1.48；V24b: 3.17→2.35）— 与"backbone变弱时T-norm有效"的V17/V21规律一致
4. **V24a的低FAR尾部更优** — FAR=0.01%时FFR=11.44%（V21为15.13%），同款融合变体1.2608%还略优于V21的1.28%；若以客户低FAR目标为准，V24a融合变体有参考价值
5. 问题手指不变：wtp_xyz_R1、btp_fys_R0、wtp_gjw_R0仍是最差三个

**建议**:
- 老协议刷分仍以 **V21 global_raw (1.1898%)** 为准
- 需向客户确认：新数据同名文件夹是否同一手指、采集条件差异（传感器/参数/贴屏）；若是同手指跨场景再采集，真正该做的是跨域评估协议+域自适应训练（如Domain-Specific BN），而非简单混数据
- 客户的降噪算法对人眼友好，但对识别训练**有害**（V24b结论），不建议在训练管线中使用

### 3.11 V25 — 同手指跨场景合并训练 (2026-06-11) ✅ 跨场景能力突破

**背景**: 客户确认 ysjz 新数据中 dy_R*/lwh_R*/zyh_R* 与老数据同名类**是同一根手指**（贴屏再采集）。
这解释了 V24 掉分原因（同一手指被标成两类 = 有害监督），并引出关键问题：**跨场景泛化**。

**跨场景基线（V21冻结模型, `cross_session_baseline_v21.py`）**:
老场景注册模板(20/类, seed42) × 新场景探针(9指×~30帧, 276 genuine / 7176 impostor)
- raw 探针: **EER = 56.91%**；denoised 探针: EER = 51.26% —— **与随机猜测(50%)无异，部署场景完全失效**
- genuine 均值仅 0.02~0.04，与 impostor 无法区分

**V25 设计** (`optimize_v25_merge.py`): V21 架构/超参不变；9 个同手指folder前70帧**并入老类**训练、
后30帧做跨场景探针；12 个真正新手指（左手、yjx）作新类（27+12=39类）；
Eval A = 老协议（模板RNG与V21比特级一致），Eval B = 跨场景协议。

**结果**:

| 协议 | V21基线 | V25a (+原图合并) | V25b (+降噪合并) |
|------|---------|----------------|----------------|
| Eval A 老协议最佳EER | **1.1898%** | 1.5375% | 1.9349% |
| Eval A global_raw | **1.1898%** | 2.0413% | 2.8243% |
| Eval B 跨场景最佳EER | 56.91% (≈随机) | **5.7135%** | 11.1692% |
| Eval B global | 56.91% | 8.06% (tnorm 6.80%) | 15.90% (tnorm 11.28%) |

V25a Eval B 最佳变体 fused_dense_all_a0.5_tnorm: d'=2.91, FFR=10%→FAR=0.59%, FAR=1%→FFR=9.06%

**发现**:
1. **合并训练使跨场景识别从完全失效(57%)变为可用量级(5.7%)** —— 10倍改善，本项目部署价值最大的单次提升
2. **代价是老协议退步0.35pp**（1.19→1.54%）：跨域不变性挤占单场景判别力（稳定性-可塑性权衡）
3. **降噪版全面落败**：老协议更差(1.93% vs 1.54%)且跨场景也差一倍(11.2% vs 5.7%) —— 与V24结论一致，
   降噪丢失了跨场景匹配所需的真实脊线细节。**结论确定: 训练管线不要用降噪数据**
4. 跨场景下**融合+T-norm占优**（所有fused_tnorm变体5.7~7.1%，纯global 8.1%），与单场景（global_raw最优）相反 ——
   域偏移下局部结构匹配与分数归一化重新变得重要
5. 跨场景 per-finger 差异大：zyh(0.44~0.85) > lwh > dy(0.25~0.39)，可能与各手指两次采集的按压姿态差异有关

**模型选择建议**:
- **演示/单场景刷分**: V21 (1.19%)
- **真实部署（注册后跨场景使用）**: V25a —— V21在该场景下完全不可用
- 进一步缩小跨场景差距需要: 更多手指的跨场景配对数据（目前仅9指）、域自适应技术（DSBN/特征对齐）、
  或注册模板包含多场景样本

### 3.12 V26 — 质量感知 + 跨场景增广 (2026-06-11) ⚠️ 增广无益

**动机**: V25a 把跨场景 EER 降到 5.71% 后，尝试用合成增广进一步缩小跨场景域差，并用质量过滤剔除新数据垃圾帧。

**先验**: 手工方向场/脊线已验证无效（见 §方向验证简报）。本节验证"训练侧"改进。

**两个版本**（均在 V25a 配方上改，仅 raw，3 seeds，双协议评估）:
- **V26**: 激进跨场景增广（增益 0.6-1.5 + 偏置场 + gamma + 强弹性 α20 + 尺度）+ 自适应质量过滤（V25a 嵌入组内一致性，丢 260 帧）
- **V26b**: 温和增广（增益 0.8-1.2 + 温和偏置场，去 gamma/强弹性）+ **关闭**质量过滤（隔离增广效果）

| 指标 | V21 | V25a | V26(激进+过滤) | V26b(温和) |
|------|-----|------|---------------|-----------|
| Eval A 老协议最佳 | 1.19% | 1.5375% | 1.9917% | **1.4358%** |
| Eval B 跨场景最佳 | 57%(失效) | **5.7135%** | 14.0677% | 7.8177% |

**发现**:
1. **合成跨场景增广不能改善跨场景** — V26 激进版灾难性退步(14%)，V26b 温和版仍比 V25a 差(7.82% vs 5.71%)。增广只是改变了哪些手指好/坏(高方差洗牌)，无净收益。真实跨场景配对(V25a 的 9 指)才是有效信号，合成扰动反而稀释它。
2. **温和增广小幅改善单场景** — V26b Eval A=1.44% < V25a 1.54%（但仍不如纯 V21 的 1.19%）。
3. **没有单一模型同时最优** — 单场景: V21(1.19%) > V26b(1.44%) > V25a(1.54%); 跨场景: V25a(5.71%) > V26b(7.82%)。**部署看跨场景 → V25a 仍是部署最优。**
4. 质量过滤在 V26 中与激进增广耦合，无法干净归因，但 V26b(无过滤)反而更好，说明过滤并非增益来源。

**结论**: 训练侧合成增广方向到此为止(激进+温和均已试)。跨场景 ~5.7% 接近"9 指跨场景配对 + 27 基类"数据量的天花板。**剩余杠杆: ①更多真实跨场景配对数据(最大，需客户) ②DSBN 域自适应(模型侧，payoff 不确定)。**

### 3.13 跨模型融合(ensemble) (2026-06-11) ✅ 单场景新纪录 + 稳定性飞跃

**动机**: 手工特征/增广/ResNet34多样性均证伪后, 尝试零训练的跨模型融合——把已训的不同配方模型(V17/V18b/V21/V25a/V26b)分数/嵌入融合, 利用误差去相关压低EER。

**方法**: 各模型512维全局嵌入(双极性TTA后)归一化→选定模型集加权平均→重归一化→余弦匹配; 可选T-norm。

**单场景结果(3次模板采样均值±std, 老27类)**:

| 配置 | EER | std | 说明 |
|------|-----|-----|------|
| V21单独 raw | 1.64% | ±0.37% | 随模板剧烈波动 |
| V21单独 tnorm | 1.58% | ±0.19% | |
| **V21+V18b + T-norm** | **1.10%** | **±0.006%** | ★ 稳定且最优 |
| V21+V18b+局部 | 1.29% | ±0.27% | 局部不稳健 |
| V21+V18b+V21r34 | 1.73% | — | ResNet34拖累 |
| score-level融合 | 1.15-1.25% | — | 不如嵌入级 |

**发现**:
1. **V21+V18b 全局嵌入融合 + T-norm = 1.10%, 刷新单场景纪录(原V21 global_raw 1.19%)**, 零训练成本。
2. **稳定性飞跃是更大价值**: V21单独随模板选择在1.2~2.1%剧烈波动(±0.37%), 融合后稳定在1.10%(±0.006%)。部署中模板是固定注册的, 这种鲁棒性极有价值。
3. **T-norm在融合上重新有效**(融合后原始分数分布更需归一化), 与V17规律一致、与V21单模型规律相反。
4. **无效的融合变体**: 局部描述子融合(不稳健)、分数级融合(不如嵌入级)、TTA旋转(单场景有害)、ResNet34成员(过拟合拖累)、加更多模板(无益)。

**跨场景结果(多采样)**: V25a+V26b+V21融合 tnorm = **6.3%±0.6%**(n=10)/6.9%(n=20), 与V25a的5.71%在噪声带内, **无稳健增益**(早期出现的5.18%/4.59%经多采样证实是采样噪声幸运低值)。跨场景276对genuine的小测试集导致EER方差~±0.6pp, 这条要诚实标注。

**结论**: 融合给出**单场景唯一的稳健新增益(1.19%→1.10%, 且大幅提升稳定性)**; 跨场景已达数据天花板, 融合无稳健改善。

### 3.14 ResNet34 多样性集成 (2026-06-11) ⚠️ 过拟合失败

V21配方换ResNet34骨干(21.7M参数)重训(models_v21r34): 单体单场景=2.27%(远差于ResNet18的1.19%, 27类上过拟合, 印证V16 ConvNeXt教训); 加入V21+V18b集成后EER从1.10%升到1.73%(误差去相关不足, 拖累集成)。**ResNet34多样性路线判死, 未训跨场景版v25r34。**

## 4. 核心发现与经验教训

### 4.1 有效的技术（按贡献排序）

| 技术 | EER改进 | 额外开销 | 推荐度 |
|------|---------|---------|--------|
| T-norm分数归一化 | -0.58% (V14→V17) | 零 | ★★★★★ |
| 注意力辅助ArcFace多任务训练 | -0.55% (V18b→V21 global) | 训练时间 | ★★★★★ |
| SWA权重平均 | -0.28% | 零 | ★★★★★ |
| 位置对比学习(局部描述符) | -0.31% (V17→V18b) | 训练时间 | ★★★★ |
| Label Smoothing (0.1) | ~-0.1% | 零 | ★★★★ |
| 全局-局部融合 | ~-0.07% | 推理时 | ★★★ |
| 注意力选点(局部匹配) | dense改善~2% | 推理时 | ★★★ |
| 3模型集成 | ~-0.2% | 训练3x | ★★★ |

### 4.2 无效/负面的技术

| 技术 | 结果 | 原因分析 |
|------|------|----------|
| ConvNeXt-Tiny (28.6M) | 退步 | 27类小数据集过拟合 |
| Gabor滤波器/GeM/多尺度 | 无改善 | CNN自学、数据不足 |
| Quality模板筛选 | 损害T-norm | 降低模板多样性 |
| 未训练的dense匹配 | 几乎无改善 | 全局特征不适合局部判别 |
| RANSAC几何验证 | 无改善 | 14×13网格太粗，impostor也67内点 |
| 去layer4+注意力池化 | 退步0.26% | 3M参数容量不足 |
| T-norm (V21) | 退步0.47% | 改进backbone后原始分数已well-calibrated |

### 4.3 关键结论

1. **多任务学习是最有效的模型改进手段** — V21的辅助注意力ArcFace比任何单一架构改动都有效
2. **简单架构 + 强正则化 > 复杂架构** — ResNet-18 (11.4M) 是最优容量
3. **后处理需要适配** — T-norm在弱模型上有效(V17)，在强模型上反而有害(V21)
4. **全局嵌入质量是基础** — layer4+GAP不可替代(V20教训)
5. **RANSAC不适合小图粗网格** — 需要大图+注意力选点(V19教训)
6. **问题手指稳定存在** — wtp_xyz_R1、btp_fys_R0始终最差
