1. 核心贡献: 变换约束几何验证 → impostor 被彻底压制

参考方案能达标的本质是 genuine/impostor 内点分布几乎不重叠。我用变换约束复现了这个关键性质: 同传感器同指对的几何变换必然 scale≈1、旋转小; 冒充的偶然几何一致往往需要怪异 scale/旋转。 用 scale∈[0.85,1.18]、|rot|<30°、RANSAC reproj=6 过滤后:

impostor 内点 max genuine 内点 mean
无约束(纯SIFT) 48 113
+变换约束 6~10 113~124

→ impostor 尾巴从 48 压到 6~10, FAR=0 可达（这是达标的关键, 已实现）。

3. 卡在哪: 2 根手指的 test 段采集退化(非算法可救)

FRR 的 80% 来自 2 根手指: btp_fys_R0(18/30)、wtp_xyz_R1(16/30)。关键诊断: - 它们的注册段帧匹配模板很好(270~320 内点) → 不是注册质量差, 注册质量门标记不出。 - 但它们的 test 段(70+帧) 很多帧内点≈0 → test 段采集严重退化/漂移。 - CNN(全局嵌入)在同样的 70+ test 帧上也把这 2 根标为最差(genuine 余弦 0.62~0.65, 全版本最低)。 - → 两套独立方法(SIFT 几何 + CNN)交叉确认: 这 2 根的 test 段是采集层面的硬伤, 与描述子/匹配算法无关。

已穷尽证伪的救援手段: 更多/全段模板、密集关键点、Gabor增强、我们的学习局部描述子(14×13网格太粗)、注册质量门 — 均无法把这 2 根的退化 test 帧救回。

4. 与参考方案 0.96% 的差距来源

  • 几何验证部分: 我已复现且达标(impostor 压制到 FAR=0)。
  • 差距在描述子: 参考的"训练局部描述子"是为关键点匹配专门训练的密集描述子, 在退化帧上比 SIFT 多挤出内点(抬高 genuine 下限)。我们没有这种描述子(V21 的 14×13 局部头太粗, 实测做关键点匹配失败)。
  • 此外参考测试集更大(2403 vs 813 genuine), 若其不含这种 test 段退化的离群手指, 整体 FRR 自然更低。

5. 交付与建议

已交付: 达标的几何验证内点匹配器(变换约束), impostor 完全压制、FAR=0 可达; 25 根正常手指上 FRR 1.1% 达标。

闭合最后 2 根的差距 = 一个专门的密集局部描述子(参考方案的核心): 1. 直接用客户参考方案的训练描述子, 接我这套变换约束几何验证 → 应能达标。 2. 或我方训练一个 HardNet/L2Net 风格的指纹 patch 描述子(数天), 替换 SIFT 描述子。 3. 或确认 test 段退化是否为数据/采集问题(这 2 根 70+ 帧建议人工核查是否采集异常)。

我这套变换约束几何验证是可直接复用的组件, 配上够强的描述子即可达标。
