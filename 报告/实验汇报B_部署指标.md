# 实验汇报 B — 部署指标（贴屏数据，空间 & 时间）

**主题**：在贴屏数据集（select，今后以此为准）上实测当前模型的部署空间与时间，对照部署预算
**当前模型**：texdesc_v3（密集纹理描述子，stride 6）
**测试环境**：CPU（部署相关）+ GPU（参考）；5 指模板、每指 20 模板 = 100 模板
**日期**：2026-06

---

## 一、结论速览

| 指标 | 预算 | CPU 实测 | GPU 实测 | 结论 |
|------|------|---------|---------|------|
| AI 模型大小 | 总 <30MB | **1.23MB**（0.30M 参数） | 同 | ✅ 远达标 |
| 单模板注册时间 | <200ms | 平均 149ms / 最慢 368ms | 平均 16ms | ⚠️ CPU 平均达标、最慢超；GPU 达标 |
| **5 指模板存储（100 模板）** | **<10MB** | **51.0MB** | 同 | ❌ **超 5.1 倍** |
| **5 指解锁（平均）** | 最快<50 / 平均<100ms | **265ms** | **136ms** | ❌ **超 2.6 倍** |

![部署指标 实测vs预算](部署指标_实测vs预算.png)

**两项硬超标：模板存储、解锁时间**；模型大小、注册时间基本达标。

---

## 二、超标项拆解（原因）

**1. 模板存储 51MB（密集描述子占用）**
- 每模板 ≈ 896 个关键点 × 128 维 float32 = 459KB；× 100 模板 = 45.9MB；加坐标 + 前景 mask = 51MB。
- 瓶颈在描述子本身的密集 + float32 + 高维。

**2. 解锁 265ms（CPU）/ 136ms（GPU）**
- 拆分：探针特征提取 CPU 147ms / GPU 13ms；匹配 100 模板 ≈ 118-126ms（单次几何匹配 ≈1.2ms × 100）。
- 瓶颈：① CPU 上特征提取慢；② 逐一匹配 100 模板（线性扫描）。

---

## 三、贴屏数据集说明

- select 数据共 **25 组手指**（dy/gyh/lwh/SSH/yjx/zyh 的 L/R 指），110×100 灰度，每组 34-100 帧。
- 本次按要求只测**空间/时间**；**精度（FAR/FRR）尚未在贴屏数据上测**（见第五节）。

---

## 四、优化方向（达标可行，预估收益）

**空间 51MB → <10MB：**
| 手段 | 收益 |
|------|------|
| 描述子 int8 量化 | ÷4 → 11.5MB |
| 降采样密度 stride 6→8 | ÷1.8 |
| 描述子降维 128→64 | ÷2 |
| 不存完整 mask（改紧凑表示） | 省 4.4MB |
| **组合（stride8 + int8 + 64维）** | **→ ~3-6MB 达标**（略损精度，需权衡） |

**解锁 265ms → <100ms：**
| 手段 | 收益 |
|------|------|
| **全局嵌入预筛**（先粗筛 top-5~10 模板，只对候选做几何匹配） | 匹配 118ms → ~10ms |
| 关键点减少（stride 8） | 特征提取 + 匹配都更快 |
| 近似最近邻（FLANN）/ "最快解锁"首个命中即返回 | 进一步降 |
| NPU / 定点化部署 | 特征提取大幅加速 |

## 五、待办与提醒

1. **精度未在贴屏数据上验证**：当前描述子在**老数据（无贴屏/不贴屏）**上训练，贴屏是不同域；建议在贴屏数据上跑一遍 FAR/FRR，若因域差下降则需在贴屏数据上微调描述子。
2. **优化与精度的权衡**：空间/时间优化（降采样、降维、量化）会轻微影响精度；需在贴屏数据上联合调参，使部署指标与精度同时达标。

---

## 六、建议下一步

1. 在贴屏数据上测精度（FAR/FRR），确认域差影响。
2. 着手部署优化：int8 量化 + stride 调整 + 全局嵌入预筛，重测部署指标至达标。
3. 若精度因域差下降，在贴屏数据上微调/重训描述子。
