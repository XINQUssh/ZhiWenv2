## 〇、总目标与三条主线

**核心目标**：FAR < 0.002%（1/5 万）时 FRR < 3%。客户偏好纯纹理方向。

| 数据域 | 最佳结果 | 是否达标 |
|--------|---------|---------|
| **老数据（27 类，单场景）** | 纹理法 FAR=0 时 FRR **2.34%**（全 27）/ **0.64%**（25 干净指） | ✅ 达标 |
| **贴屏（ysjz 跨场景再采集）** | 降噪 V2 + 5 帧合并模板 EER **10.1%**；严格 1/5 万仍不达标 | ❌ 未达标 |
| **光学（FVC DB1_A）** | center 裁剪 EER **13.7%**（跨域，best.pth 未重训） | ❌ 但证明重叠是关键 |

**贯穿全程的核心认知**：
1. **EER 优 ≠ 严格 FAR 达标**：全局嵌入 EER 1.1% 但连续分数尾部重叠，FAR<0.002% 处 FRR≈100%；严格 FAR 需要"内点几何"这类离散可分分布。
2. **严格 FAR 的瓶颈是冒充尾巴**（不同手指偶然凑出的几何一致内点），不是描述子强弱。
3. **匹配成败本质取决于两次采集的重叠**（光学 center vs random 受控实验坐实）。

## 2.11 结合指纹脊线特征的多级融合（2026-07-07）
- 背景：客户提出当前学习描述子"太通用"（放到其它纹理图也可行），应结合指纹脊线特征（参考《基于多级特征融合的部分指纹识别》刘超）。
- 实现（脚本 `fp_fusion.py`）：**Level1 学习纹理描述子 + Level2 脊线方向场一致性 + Level3 脊线频率一致性**——对齐后过滤"不遵守脊线流向/频率"的巧合匹配。
- **btp（9 指）**：基线已很好（gen 196 内点、EER 3.3%、FAR=0 FRR 3.3%，那 3.3% 是某退化指拖累）；融合把冲突均值 3.06→2.46、genuine 几乎不掉（196→194）——验证约束不误伤真匹配，但 btp 已太干净无尾巴可治。
- **贴屏（25 指）**：冲突均值 3.29→2.83，但 **冲突 max 26.9 不变 → FAR=0 不变**。根因：最坏冒充对（不同手指小区域脊线局部相似）是"指纹可信"的，逐点方向/频率挡不住。
- 结论：事后融合让方法**指纹专属、不误伤真匹配**，但压不动最坏冲突。更强方向：**全局方向场相关**（整块脊线流向相关）/ **特征级融合**（方向、频率作输入通道，训练"指纹感知描述子"）。

# 第四阶段：部署优化

## 4.1 空间（达标）
- int8 量化描述子 + stride6→8 + 紧凑 mask（÷4）+ 全局嵌入预筛。**100 模板存储 51MB → 6.95MB（<10MB）**；模型 1.23MB。脚本 `optimized_matcher.py` / `benchmark.py`。

## 4.2 时间
- **GPU 达标**：单模板注册 ~21ms、5 指解锁平均 32ms/最快 23ms。
- **CPU 特征提取是大头**（~300ms/帧，含/不含均如此），若把 feat 计入则 CPU 超预算，需 NPU/GPU 或更轻特征。
- **仅匹配（不含 feat，客户口径）**：单次几何匹配 ~0.51ms；100 模板早退解锁 **最快 0.5ms / 平均 6–8ms / 全量 51ms**；5×10=50 模板早退 最快 0.5ms/平均 6ms。**均远达标**。脚本 `cpu_timing.py` / `cpu_threads.py` / `cpu_match_time.py` / `unlock_sim.py` / `deploy_eval.py`。

## 4.3 描述子长度 / 内存
- 单局部描述子 128 维 → int8 1024 bit / 二值化 128 bit；**每模板含全部局部 ≈ 63KB（504 关键点）**。纹理法为纯局部，无单独全局；<512 bit 只有紧凑全局描述子能满足。
- 内存：1.23MB 是模型权重非运行内存；运行 RAM ≈ 模型 1.2 + 模板 6.95 + 缓冲 3 ≈ 11MB（< 30MB）。
- 报告 `补充实验_CPU耗时与模板数量.md`。

# 五、核心结论与建议

1. **老数据单场景已达标**（纹理法 FAR=0 时 FRR 2.34%/25 根 0.64%）。
2. **贴屏（跨场景）严格 1/5 万仍不达标**，最好 EER 10.1%（降噪 V2 + 5 帧合并模板）。已验证**算法侧探到头**：描述子重训（贴屏+光学各 2 轮）、几何验证松紧、量化、降噪/CLAHE 均不改变净精度天花板。
3. **有效杠杆排序**：① **5 帧合并模板（EER 砍半）** ② **降噪 V2 方向** ③ **采集提清晰度/重叠/减形变**（贴屏失败 55% 模糊、28% 扭曲、17% 重叠；光学 center vs random 证明重叠决定一切）。
4. **严格 1/5 万的最后瓶颈=冒充尾巴**（不同手指偶然凑内点），需更干净的采集 + 更强的几何/描述子区分度，或放宽工作点。
5. **部署**：空间达标（6.95MB）；时间按"仅匹配"口径达标（最快 0.5ms/平均 6ms）；feat 需 NPU/GPU。

**建议下一步**：数据侧持续用 **V2 降噪 + 5 帧合并模板 + 保证帧间重叠/清晰**；每版数据用附带的 `fp_test.py` 自测横比；算法侧待更多干净跨场景配对数据后再评估重训。
